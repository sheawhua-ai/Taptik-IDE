# UI 设计迭代与认知对齐

## 1. 顶部操作栏与 Tabs 降噪 (App.tsx)
**方案说明：** 去掉了右上角原本存在的“常规/创作/监控/审核”专注模式 Tab。
**设计理由：** 您提到“左边的tab，和右上角的小tab是否有重叠了？”。确实，原先过多的状态切换会增加开发心智负担与视觉噪音。目前让“全域巡航、矩阵分发”等 Workflow 标签作为唯一核心的导航切换，使操作更加纯粹聚焦，不再被生硬的模式打断。

## 2. 商家项目看板 (MerchantMatrix.tsx) - 连贯性与微小指令
**方案说明：** 取消了原先简单的间距 grid 网格，采用了 `flex` 横向伸展的阶段式展现（STG 1 -> STG 5）。并且在阶段与阶段的“缝隙连接处”，加入了一个微小的串联指令按键 (Send Icon)。
**设计理由：** 遵从“保持大卡片高密度、阶段连贯式的展现统治地位”。这让项目进度的“流（Flow）”概念在视觉上得到统一，而不是一块块碎裂的积木。卡片间的小按钮既保证了“极简”的观感，又巧妙地在任务转移（交接棒）的节点赋能给子智能体（Sub-agent）进行指令穿透。

## 3. 智造工坊 (ContentProduction.tsx) - 降级机械感
**方案说明：** 
- **移除无用项：** 移除了“发布目标平台”模块；置灰并禁用了“视频分镜”。
- **流水线重构：** 将原先那种“做完一步隐藏一步、像填表一样机械”的硬切形式，重构为**纵向时间流管道展开式**。用户完成上一步后，步骤卡片不下卷，而是在原位下移变淡（opacity-70），同时用连贯的连接线平滑引出下一个节点。
**设计理由：** 流水线本质上是“过程累积”的体系。旧设计存在“阶段间断崖式割裂感”。新设计更像一个持续运行的对话管道，让 AI 介入的中间思考过程与人类控制权无缝咬合，视觉上行云流水但不丧失每一步的掌控感。

## 4. 全域巡航 (Strategy) 的重构与认知降噪
**方案说明：**
- **移除冗余子选项卡：** 完全删除了原先内部嵌套的“运营策略中心”、“蓝海词雷达”和“平台资产矩阵”三个 tabs，直接将功能整合为统一的全局画布。
- **构建联动体系：** 引入了“当前活跃项目联合策略 (Data-Driven Blueprint)” 的概念。向用户传达一个心智：**策略是基于项目的**。当在左侧控制台确定了项目方向后，系统会自动整合“外部全域蓝海雷达”的数据与“内部归因模型库”的历史 ROI 反馈，生成高置信度的决策方向。
- **聚焦单点输出：** 过去的“蓝海词雷达”像一个孤立的看盘工具，现在它变成了“策略探测输出”的 Pipeline 环节。推荐出来的蓝海词，不仅有综合机会指数，更有明确的“送入智造工坊”长剑串联按钮，构成闭环。

**设计理由：** 当一个商家的操作矩阵不断膨胀时，工具栈的堆砌感是致命的体验杀手。“蓝海雷达”和“资产矩阵”如果作为独立工具孤悬在外，它是不产生生产力的。通过这种整合，全域巡航从“仪表盘”变成了指挥“执行流水线”的发令枪，不仅大幅降低了信息噪音，也完全厘清了数据与动作的边界。
