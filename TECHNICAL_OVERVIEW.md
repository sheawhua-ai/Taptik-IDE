# TAPTIK Technical & Strategy Overview

本文档提供了 TAPTIK 平台的完整技术蓝图、功能架构及核心交互逻辑，旨在为深度系统分析（如 DeepSeek 分析）提供全面背景，以识别现有实现与业务愿景之间的差异。

---

## 1. 业务愿景与产品定义 (Product Vision)

**TAPTIK** 定位于“面向商家的 AI 运营全流程闭环平台”。它不是一个简单的对话框，而是一个将 AI 能力（Skills）、项目管理、自动化流水线（Pipeline）与业务结果（ROI）深度整合的 SaaS 系统。

### 核心价值主张：
- **能力原子化**: 将复杂的 AI 任务（如视觉去重、蓝海词挖掘）封装为名为 "Skill" 的独立资产。
- **流程闭环化**: 从发现机会（数据）到生成内容（AI），再到分发与归因，全链路跑通。
- **开发者生态**: 设定“500次冷启动”规则，鼓励高质量 Skill 的自主变现与分发。

---

## 2. 技术栈与架构 (Tech Stack)

### 2.1 前端核心
- **框架**: React 18 + TypeScript (严格类型模式)。
- **构建工具**: Vite (极速热更新)。
- **样式方案**: Tailwind CSS (JIT 编译器，极致圆润设计语言)。
- **图标系统**: Lucide React (业务隐喻映射)。
- **动效库**: Framer Motion (用于路径切换、模态框、列表入场动效)。

### 2.2 状态管理与数据结构
- **本地存储**: 当前采用 React 响应式 State 管理，关键项目数据存储在 `MOCK_PROJECTS` 常量中。
- **核心实体**:
    - `Project`: 包含文件树（FileTree）、历史会话、品牌物料等。
    - `Skill`: 包含计费模型（价格/次/月）、业务分类、调用统计、准入门槛（Thresholds）。
    - `Pipeline`: 定义端到端的自动化环节与实时连通性状态。

## 3. 核心功能模块 (Functional Modules)

### 3.1 AI 协作室 (AI Workspace)
- **多项目隔离**: 支持不同商家的业务隔离（如宠物食品组 vs. 美妆旗舰店）。
- **智能提示系统 (Mentions)**: 
    - 输入 `@`：唤起 Skill 能力列表，进行显式调用。
    - 输入 `/`：唤起 Agent 模式选择。
- **Agent 决策建议**: 系统自动监测上下文（如原创度低），通过对话流嵌入“付费/免费 Skill 推荐卡片”，引导用户安装能力。
- **文件与 RAG 对齐**: 侧边栏支持直接预览项目文件夹、RAG 知识库及会话历史。

### 3.2 能力增强中心 (Skill Market)
- **双态显示**: “我的能力轴”（开发者视角）与“能力商店”（使用者视角）。
- **冷启动规则 (Monetization Logic)**: 
    - 强制免费期：Skill 需累计调用 500 次方可定价。
    - 视觉引导：动态扫光进度条展示达成百分比（如 8.4%）。
- **Skill 蒸馏器 (Builder)**: 支持通过自然语言描述或上传 `.md` 专家指令集来快速构建并部署 Skill。

### 3.3 业务链路看板 (Pipeline & Dashboard)
- **Health Check**: 实时展示“蓝海词挖掘 -> 内容生成 -> 路由 -> 写入 -> 归因”的全链路健康度。
- **待办清单**: 将技术瓶颈（如 API 500 错误）翻译为具体的业务动作（如“修 DeepSeek 直连”）。
- **全链路 ROI**: 整合实时数据回收，不依赖商家登录态进行穿透分析。

## 4. 设计语言与交互规范 (UX/UI DNA)

- **极致圆润**: 采用 `rounded-[32px]` 到 `rounded-xl` 的分层圆角。
- **业务隐喻图标 (Business Icons)**: 拒绝通用 AI 星标，使用 `Filter` (挖掘)、`Layers` (矩阵)、`Dna` (去重) 等具有明确业务指向的视觉符号。
- **技术感增强**: 对于系统路径（如 `exports/`）和 API 动作，使用 `font-mono` + 背景色块进行语义化处理。

---

## 5. 当前实现状态与缺项分析 (Current Status & Gaps)

### 5.1 已实现 (Completed)
- 核心 UI 框架与导航交互。
- Skill 市场的浏览、详情查看及构建流程预览。
- 业务仪表盘（Pipeline Health Check）的视觉模型。
- AI 协作室的智能卡片推荐逻辑。

### 5.2 待实现/潜在缺项 (Gaps to identify)
1. **多模态数据闭环**: 如何真正实现从 Excel 到代码变现的自动化修改链路？
2. **多层级权限模型**: 针对品牌主、运营实习生、ISV 开发者的颗粒度差异。
3. **真实 RAG 接入**: 知识库的向量化存储与检索增强的物理层实现。
4. **变现路径细化**: 500 次调用后的自动化计费逻辑与账户体系挂钩。
5. **离线与同步**: 本地 Tauri 巡检与云端逻辑的一致性同步。
